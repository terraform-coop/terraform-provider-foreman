foreman_domain¶

Description¶

Foreman representation of domain. Domains serve as an identification string that defines autonomy, authority, or control for a portion of a network.

Interactivity¶

  • Creatable: YES
  • Deletable: YES
  • Mutable: YES

Example Usage¶

resource "foreman_domain" "example" {

  name = "dev.dc1.company.com"
}

Argument Reference¶

The following arguments are supported:

  • fullname - (Optional; schema.TypeString) Description of the domain
  • name - (schema.TypeString) The name of the domain - the full DNS domain name.

Attributes Reference¶

The following attributes are exported:

  • fullname - (schema.TypeString) Description of the domain
  • name - (schema.TypeString) The name of the domain - the full DNS domain name.
