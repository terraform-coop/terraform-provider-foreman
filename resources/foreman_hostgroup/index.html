foreman_hostgroup¶

Description¶

Hostgroups are organized in a tree-like structure and inherit values from their parent hostgroup(s). When hosts get associated with a hostgroup, it will inherit attributes from the hostgroup. This allows for easy, shared configuration of various hosts based on common attributes.

Interactivity¶

  • Creatable: YES
  • Deletable: YES
  • Mutable: YES

Example Usage¶

resource "foreman_hostgroup" "example" {





  name = "compute"










}

Argument Reference¶

The following arguments are supported:

  • architecture_id - (Optional; schema.TypeInt) ID of the architecture associated with this hostgroup.
  • compute_profile_id - (Optional; schema.TypeInt) ID of the compute profile associated with this hostgroup.
  • domain_id - (Optional; schema.TypeInt) ID of the domain associated with this hostgroup.
  • environment_id - (Optional; schema.TypeInt) ID of the environment associated with this hostgroup.
  • medium_id - (Optional; schema.TypeInt) ID of the media associated with this hostgroup.
  • name - (schema.TypeString) Hostgroup name.
  • operatingsystem_id - (Optional; schema.TypeInt) ID of the operating system associated with this hostgroup.
  • parameters - (Optional; schema.TypeMap) A map of parameters that will be saved as hostgroup parameters in the group config.
  • parent_id - (Optional; schema.TypeInt) ID of the parent hostgroup.
  • ptable_id - (Optional; schema.TypeInt) ID of the partition table associated with this hostgroup.
  • puppet_ca_proxy_id - (Optional; schema.TypeInt) ID of the smart proxy acting as the puppet certificate authority server for this hostgroup.
  • puppet_proxy_id - (Optional; schema.TypeInt) ID of the smart proxy acting as the puppet proxy server for this hostgroup.
  • pxe_loader - (Optional; schema.TypeString) Operating system family. Values include: "None", "PXELinux BIOS", "PXELinux UEFI", "Grub UEFI", "Grub2 UEFI", "Grub2 UEFI SecureBoot", "Grub2 UEFI HTTP", "Grub2 UEFI HTTPS", "Grub2 UEFI HTTPS SecureBoot", "iPXE Embedded", "iPXE UEFI HTTP", "iPXE Chain BIOS", "iPXE Chain UEFI"
  • realm_id - (Optional; schema.TypeInt) ID of the realm associated with this hostgroup.
  • root_password - (schema.TypeString) Default root password
  • subnet_id - (Optional; schema.TypeInt) ID of the subnet associated with the hostgroup.

Attributes Reference¶

The following attributes are exported:

  • architecture_id - (schema.TypeInt) ID of the architecture associated with this hostgroup.
  • compute_profile_id - (schema.TypeInt) ID of the compute profile associated with this hostgroup.
  • domain_id - (schema.TypeInt) ID of the domain associated with this hostgroup.
  • environment_id - (schema.TypeInt) ID of the environment associated with this hostgroup.
  • medium_id - (schema.TypeInt) ID of the media associated with this hostgroup.
  • name - (schema.TypeString) Hostgroup name.
  • operatingsystem_id - (schema.TypeInt) ID of the operating system associated with this hostgroup.
  • parameters - (schema.TypeMap) A map of parameters that will be saved as hostgroup parameters in the group config.
  • parent_id - (schema.TypeInt) ID of the parent hostgroup.
  • ptable_id - (schema.TypeInt) ID of the partition table associated with this hostgroup.
  • puppet_ca_proxy_id - (schema.TypeInt) ID of the smart proxy acting as the puppet certificate authority server for this hostgroup.
  • puppet_proxy_id - (schema.TypeInt) ID of the smart proxy acting as the puppet proxy server for this hostgroup.
  • pxe_loader - (schema.TypeString) Operating system family. Values include: "None", "PXELinux BIOS", "PXELinux UEFI", "Grub UEFI", "Grub2 UEFI", "Grub2 UEFI SecureBoot", "Grub2 UEFI HTTP", "Grub2 UEFI HTTPS", "Grub2 UEFI HTTPS SecureBoot", "iPXE Embedded", "iPXE UEFI HTTP", "iPXE Chain BIOS", "iPXE Chain UEFI"
  • realm_id - (schema.TypeInt) ID of the realm associated with this hostgroup.
  • root_password - (schema.TypeString) Default root password
  • subnet_id - (schema.TypeInt) ID of the subnet associated with the hostgroup.
  • title - (schema.TypeString) The title is the fullname of the hostgroup. A hostgroup's title is a path-like string from the head of the hostgroup tree down to this hostgroup. The title will be in the form of: "//.../".
